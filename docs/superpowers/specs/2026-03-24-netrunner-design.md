# NETRUNNER — 게임 디자인 스펙

**날짜:** 2026-03-24
**장르:** 카드 덱빌딩 로그라이크
**플랫폼:** GameHub (biotech-tycoon Next.js 프로젝트)
**상태:** 승인됨

---

## 1. 개요

사이버펑크 2087년, 메가코프가 지배하는 도시. 플레이어는 언더그라운드 넷러너로서 메가코프의 서버를 해킹하며 최상위 AI 보스를 무너뜨려야 한다.

**승리 조건:**
- 스토리 모드: Act 1 → 2 → 3 → 최종 보스 처치
- 엔드리스 모드: 최대한 높은 층까지 생존, 점수 기록

**패배 조건:** HP가 0이 되면 게임오버 (해당 런 종료)

---

## 2. 클래스 시스템

플레이어는 게임 시작 시 3개 클래스 중 하나를 선택한다.

| 클래스 | 플레이 스타일 | 시작 HP | 시작 덱 특징 |
|---|---|---|---|
| 🗡️ **Ghost** | 공격형 — 빠른 단타 콤보 | 75 | 저코스트 공격 카드 위주 |
| 🛡️ **Tank** | 방어형 — 방어막 쌓고 반격 | 100 | 블록 + 가시 피해 위주 |
| ⚡ **Hacker** | 버프/디버프형 — 카드 조작 | 70 | 상태이상 + 드로우 위주 |

각 클래스는 전용 카드 풀을 보유하며, 공용 카드(무색)도 존재한다.

---

## 3. 게임 모드

### 스토리 모드
- Act 1 → Act 2 → Act 3 → 최종 보스
- 각 Act는 노드 맵으로 구성
- 총 플레이 시간 약 40~60분

### 엔드리스 모드
- 층이 무한히 이어짐
- 진행할수록 적 강화
- 최고 기록(층수 + 점수) localStorage 저장

### 세이브 시스템
- Zustand `persist` 미들웨어 + localStorage
- 슬롯: `netrunner-story`, `netrunner-endless` (모드별 분리)
- 노드 이동 후 자동 저장
- 게임 재접속 시 이어하기 제공

---

## 4. 맵 구조

각 Act마다 노드 맵이 랜덤 생성된다. 플레이어는 연결된 경로를 따라 노드를 선택하며 이동한다.

| 노드 타입 | 비율 | 내용 |
|---|---|---|
| ⚔️ 전투 | 45% | 일반 적과 카드 전투 |
| ❓ 이벤트 | 20% | 텍스트 선택지 (리스크/리워드) |
| 🏪 상점 | 15% | 카드 구매/제거, 유물 구매 |
| 💀 엘리트 | 10% | 강한 적, 희귀 유물 보상 |
| 🔥 보스 | 1개/막 | 막 마지막 노드, 보스 전투 |

맵은 가로 7열 × 세로 15행(Act당) 격자에서 생성되며, 시작 노드에서 보스 노드까지 연결된 경로만 선택 가능하다.

## 5. 전투 시스템

### 턴 흐름
```
턴 시작 → 카드 5장 드로우 → 에너지 3 충전
→ 카드 플레이 (에너지 소모) → 턴 종료 → 적 행동 실행
→ 반복 → 적 HP 0 = 승리 / 플레이어 HP 0 = 게임오버
```

### 핵심 메카닉

| 개념 | 설명 |
|---|---|
| **에너지** | 매 턴 3 충전, 카드마다 소모량 상이 |
| **블록** | 피해를 흡수하는 방어막, 턴 종료 시 초기화 |
| **상태이상** | DoT/디버프 효과, 턴마다 카운터 감소 |
| **유물** | 전투 내내 또는 영구 적용되는 패시브 아이템 |

### 상태이상 종류

| 이름 | 효과 |
|---|---|
| 🔴 **출혈** | 매 턴 X 피해 후 카운터 1 감소 |
| ⚡ **감전** | 블록을 쌓지 못함 |
| 🔒 **잠금** | 다음 턴 카드 1장 사용 불가 |
| 💀 **과부하** | 다음 행동 시 추가 피해 발생 |

### 적 AI
- 적은 매 턴 다음 행동을 아이콘으로 미리 표시 (공격 예고 / 방어 / 버프)
- 플레이어가 읽고 전략적으로 대응

## 7. 적 & 보스

### 일반 적 6종

**1. 경비 드론 (Guard Drone)** — 기본 입문용
- HP: 25~35 (랜덤)
- 골드: 10~15
- 패턴: ⚔️ 8 → 🛡️ 블록 5 → ⚔️ 8 순환

**2. 코퍼 에이전트 (Corp Agent)** — 버프 후 강타
- HP: 35~45 (랜덤)
- 골드: 15~20
- 패턴: 💪 버프(다음 공격 +5) → ⚔️ 14 → 🛡️ 블록 8 순환

**3. 정찰 봇 (Scout Bot)** — 약하지만 빠른 다중 타격
- HP: 20~28
- 골드: 8~12
- 패턴: ⚔️ 4 → ⚔️ 4 → ⚔️ 4 → 🛡️ 블록 3 순환

**4. 스팸 봇 (Spam Bot)** — 상태이상 도배
- HP: 30~38
- 골드: 12~18
- 패턴: 🔴 출혈 2 → ⚔️ 6 → ⚡ 감전 1 → ⚔️ 8 순환

**5. 장갑 경비 (Armored Guard)** — 높은 방어, 반격
- HP: 40~50
- 골드: 18~24
- 패턴: 🛡️ 블록 12 → 🛡️ 블록 12 → ⚔️ 20 순환
- **특수:** 블록 보유 시 공격받으면 3 반사 피해

**6. 충격 부대원 (Shock Trooper)** — 복합 상태이상
- HP: 38~48
- 골드: 16~22
- 패턴: ⚡ 감전 2 → ⚔️ 12 → 💀 과부하 1 → ⚔️ 15 순환

- C) 그냥 지나친다 → 아무 일 없음

## 10. 골드·상점·Rest 노드·엔드리스 점수

### 골드 획득 체계

| 출처 | 골드량 |
|---|---|
| 일반 전투 승리 | 10~20 (랜덤) |
| 엘리트 전투 승리 | 25~35 (랜덤) |
| 보스 처치 | 50 |
| 이벤트 (선택지에 따라) | 0~60 |

### 상점 구성

상점 방문 시 아래 항목이 랜덤으로 진열됨:

| 항목 | 수량 | 가격 |
|---|---|---|
| 카드 (Common) | 2장 | 골드 50 |
| 카드 (Uncommon) | 2장 | 골드 80 |
| 카드 (Rare) | 1장 | 골드 130 |
| 유물 (Common) | 1개 | 골드 100 |
| 유물 (Uncommon) | 1개 | 골드 150 |
| 카드 제거 | 1회 | 골드 75 (2회차 이후 +25 누적) |
| HP 회복 (30 회복) | 1회 | 골드 60 |

- 상점 재방문 시 품목 교체 없음 (판매된 것만 사라짐)
- 할인 유물 보유 시 전체 가격에 % 적용

### Rest 노드

맵에 Act당 2~3개 배치. 도착 시 2개 선택지 중 하나 선택:

| 선택지 | 효과 |
|---|---|
| 🛏️ 휴식 | 최대 HP의 30% 회복 |
| 🔧 카드 업그레이드 | 덱에서 카드 1장 선택해 업그레이드 |

업그레이드 원칙: 공격/블록 수치 카드 → 효과 수치 +50%, 파워 카드 → 코스트 -1

### 엔드리스 모드 점수 계산

```
최종 점수 = (층수 × 100)
          + (처치한 적 수 × 10)
          + (처치한 엘리트 수 × 50)
          + (처치한 보스 수 × 200)
          + (남은 HP × 2)
          + (보유 골드 × 1)
          + (보유 유물 수 × 30)
          - (덱 카드 수 × 2)   ← 덱 슬리밍 보너스
```

등급 기준:

| 등급 | 점수 |
|---|---|
| 🥉 C | 0 ~ 999 |
| 🥈 B | 1,000 ~ 2,999 |
| 🥇 A | 3,000 ~ 5,999 |
| 💎 S | 6,000 ~ 9,999 |
| 👑 SS | 10,000+ |

- localStorage 키: `netrunner-endless-ghost`, `netrunner-endless-tank`, `netrunner-endless-hacker` (클래스별 분리)

### 핵심 타입 정의

```ts
// 카드 인스턴스 (상태에 저장 — 직렬화 가능)
interface CardInstance {
  id: string          // "ghost_swift_strike"
  upgraded: boolean
}

// 카드 정의 (cards.ts에서 id로 룩업 — 함수 포함, 상태에 저장 안 함)
interface CardDefinition {
  id: string
  name: string
  cost: number
  type: "attack" | "skill" | "power"
  rarity: "common" | "uncommon" | "rare"
  classes: ("ghost" | "tank" | "hacker" | "neutral")[]
  description: string
  // 효과는 CombatAction 배열 반환 — 순서 있는 부수효과 지원
  effect: (state: CombatState, upgraded: boolean) => CombatAction[]
}

// 전투 중 카드/유물 효과 함수에 전달되는 읽기 전용 컨텍스트
interface CombatState {
  player: NetrunnerState["player"]
  currentEnemy: EnemyInstance
}

// 전투 액션 커맨드 (effect 함수 반환값 — 순서대로 실행됨)
type CombatAction =
  | { type: "deal_damage"; target: "player" | "enemy"; amount: number }
  | { type: "add_block"; target: "player" | "enemy"; amount: number }
  | { type: "apply_status"; target: "player" | "enemy"; status: StatusEffect }
  | { type: "draw_cards"; amount: number }
  | { type: "gain_energy"; amount: number }
  | { type: "heal"; amount: number }

// 상태이상
interface StatusEffect {
  id: "bleed" | "shock" | "lock" | "overload"
  stacks: number
}

// 적 인스턴스 (전투 중 상태)
interface EnemyInstance {
  definitionId: string     // enemies.ts 룩업 키
  hp: number
  maxHp: number
  block: number
  statusEffects: StatusEffect[]
  intent: EnemyIntent      // 다음 행동 미리 표시
}

interface EnemyIntent {
  type: "attack" | "defend" | "buff" | "debuff"
  value?: number           // 공격이면 피해량
}

// 맵 노드
type NodeType = "combat" | "elite" | "boss" | "shop" | "event" | "rest"

interface MapNode {
  id: string
  type: NodeType
  x: number                // 열 위치 (0~6)
  y: number                // 행 위치 (0~14)
  connections: string[]    // 연결된 노드 id 목록
  cleared: boolean
}

// 유물
interface RelicInstance {
  id: string               // relics.ts 룩업 키
}

// 유물 정의 (relics.ts)
interface RelicDefinition {
  id: string
  name: string
  description: string
  rarity: "common" | "uncommon" | "rare" | "boss"
  // 훅 기반 패시브: 전투 이벤트 발생 시 호출
  onCombatStart?: (state: CombatState) => CombatAction[]
  onCardPlayed?: (state: CombatState, card: CardInstance) => CombatAction[]
  onTurnStart?: (state: CombatState) => CombatAction[]
  onDamageReceived?: (state: CombatState, amount: number) => CombatAction[]
}
```

### 상태 구조 (Zustand)

상태에는 **직렬화 가능한 데이터만** 저장한다. 카드/유물 효과 함수는 `cards.ts`, `relics.ts`에서 id로 런타임 룩업한다.

```ts
interface NetrunnerState {
  phase: "title" | "map" | "combat" | "reward" | "shop" | "event" | "victory" | "gameover"
  mode: "story" | "endless"
  player: {
    class: "ghost" | "tank" | "hacker"
    hp: number
    maxHp: number
    gold: number
    block: number
    energy: number
    maxEnergy: number
    deck: CardInstance[]        // 전체 덱 (id만)
    hand: CardInstance[]        // 현재 패
    drawPile: CardInstance[]    // 드로우 더미
    discardPile: CardInstance[] // 버린 더미
    relics: RelicInstance[]     // 유물 (id만)
    statusEffects: StatusEffect[]
  }
  currentEnemy: EnemyInstance | null
  map: MapNode[][] | null  // Phase 1에서는 null, Phase 2부터 생성됨
  currentNode: { act: number; x: number; y: number } | null
  run: { act: number; floor: number; score: number }
  pendingRewardCards: string[]  // 보상 화면에서 선택할 카드 id 3개
}
```

### 카드 플레이 인터랙션
- **클릭으로 플레이** (기본): 카드 클릭 → 하이라이트 → 다시 클릭 또는 "플레이" 버튼으로 확정
- **드래그로 플레이** (향상): Framer Motion `drag` prop으로 플레이 존에 드래그 — 터치 지원 포함, 드롭 존은 중앙 영역(y < 화면 중간)으로 판정

Phase 1에서는 클릭 방식으로 구현하고, Phase 4 폴리시 단계에서 드래그 방식으로 업그레이드한다.

### 세이브
- `zustand/middleware`의 `persist` 사용
- localStorage 키: `netrunner-story`, `netrunner-endless`
- 함수가 포함된 `CardDefinition`/`RelicDefinition`은 상태에 없으므로 직렬화 문제 없음

### 애니메이션 (Framer Motion)
- 카드 선택 시 위로 떠오르는 효과 (y 이동)
- 카드 획득 시 flip 애니메이션 (rotateY)
- 피해/회복 숫자 floating 텍스트 (y 이동 + opacity)
- 맵 노드 이동 시 경로 하이라이트 애니메이션

## 12. 구현 우선순위

### Phase 1 — 플레이 가능한 전투 루프 (Runnable milestone)
- Zustand 스토어 + 모든 타입 정의
- Ghost 클래스 전용 카드 10장 (`cards.ts`)
- 적 2종 + 보스 1종 (`enemies.ts`) — EnemyInstance, EnemyIntent 포함
- 상태이상 4종 처리 로직
- CombatScreen (패, 에너지, 적 HP, 행동 예고, 턴 종료)
- 보상 화면 (전투 후 카드 3장 선택)
- TitleScreen (클래스 선택 — Ghost만 활성)
- 하드코딩된 선형 전투 진행 (맵 없이 전투 → 보상 → 전투 반복)
- **이 단계에서 전투 한 판을 완전히 플레이할 수 있어야 함**

### Phase 2 — 맵 & 나머지 클래스
- MapNode 타입 + 맵 생성 알고리즘 (`mapConfig.ts`)
- MapScreen 컴포넌트
- Tank / Hacker 클래스 카드 각 10장
- 상점 화면 + 이벤트 화면

### Phase 3 — 완성도
- 유물 시스템 (`relics.ts`, `RelicBar.tsx`)
- 보스 3종 완성 (Act별 1종)
- 엔드리스 모드
- Zustand `persist` 세이브 시스템

### Phase 4 — 폴리시
- 카드 드래그 앤 드롭 업그레이드
- 피해 floating 텍스트 애니메이션
- 밸런스 조정
- gamesRegistry에 등록 (플레이 가능 상태 시)
